# 개인정보 법률 검토 (P1)

> ⚠️ **법률 자문 아님.** 아래는 코드베이스 실측에 기반한 **기술·컴플라이언스 검토**다.
> 특히 "만 14세 미만" 항목은 법적 판단이 필요하므로 최종적으로 변호사/전문가 확인을 권한다.
> 검토 기준: 대한민국 「개인정보 보호법」(PIPA). EU 사용자 대상 서비스가 아니므로 GDPR은 참고 수준.

## 0. 결론 한 줄

TypingVerse는 **실명·이메일·전화·주소·비밀번호·생년월일·IP·결제정보를 일절 수집하지 않는** 가명·최소수집 서비스다.
제3자 트래커(GA·Sentry·픽셀 등)도 **0개**다. 따라서 컴플라이언스 부담은 낮으나, 현재 **개인정보처리방침 페이지가 없는 것**이 가장 큰 공백이며, **국외 저장(Cloudflare)·보존기간·만 14세 미만** 세 가지를 방침에 명시·결정해야 한다.

---

## 1. 수집·처리 인벤토리 (실측)

### 1-1. 서버 저장 (Cloudflare D1, `schema.sql`)

| 항목 | 수집 시점 | 성격 | 목적 |
|---|---|---|---|
| `users.id` (서버 발급 UUID) | 가입 | 가명 식별자 | 계정 식별 |
| `users.nickname` (2–12자, 한글/영숫자) | 가입 | 이용자 입력(실명 아님) | 리더보드·표시 |
| `users.avatar` (프리셋 예: `cat`) | 가입 | 비식별 | 표시 |
| `users.grade_band` (`elem`/`middle`/`high`/`adult`) | 가입(선택) | **연령대 범주(정확 나이 아님)** | 학년별 리더보드 필터 |
| `users.created_at` / `last_seen_at` | 자동 | 접속 시각 | 운영 |
| `devices.device_hash` | 가입 | **deviceId의 HMAC — 원문 저장 안 함** | 기기-계정 매칭·복구 |
| `scores` / `challenges` | 플레이 | 타이핑 기록(속도·정확도) | 통계·랭킹 |
| `wallets` / `wallet_transactions` | 플레이 | **가상 재화(코인·XP) — 현금 아님** | 게임 보상 원장 |
| `league_*` | 주간 | 리그 순위·정산 | 승강제 |
| `classes` / `class_members` / `transfer_codes` | 선택 | 학급 그룹·계정 이전 코드 | 교실 기능 |

**미수집(스키마에 컬럼 없음):** 이메일, 실명, 전화번호, 주소, 비밀번호, 생년월일(정확), IP 주소.

### 1-2. 클라이언트 저장 (브라우저 localStorage)

`typingverse-device`(기기ID), `typingverse-token`(인증 토큰), `typingverse-user`(프로필),
`typingverse-settings`·`-progress`·`-stats`·`-sessions`·`-game-results`·`-quests`·`-mascot`·`-challenge`·`-theme`.

- 전부 **1인칭 로컬 데이터**이며 제3자로 전송되지 않는다(자체 API `/api/*` 외 아웃바운드 호출 없음).
- 쿠키 미사용(모든 상태는 localStorage). → **쿠키 동의 배너 불필요**.

### 1-3. 제3자 처리자 / 국외 이전

- **Cloudflare Pages + D1**: 서버·DB가 Cloudflare 인프라(대한민국 외)에 저장된다 → **개인정보 국외 이전에 해당**(PIPA §28-8). 방침에 고지 필요.
- 광고·분석 SDK, 소셜 로그인, 결제 PG: **없음**.

## 2. PIPA 관점 갭 분석

| 조항/원칙 | 현재 상태 | 조치 |
|---|---|---|
| §3 최소수집·목적제한 | ✅ 매우 잘 준수(가명·최소) | 유지 |
| §30 **개인정보처리방침 공개 의무** | ⛔ **페이지 없음** | **P1: 방침 작성·공개**(부록 초안) |
| §28-8 국외이전 고지 | ⚠️ 미고지 | 방침에 Cloudflare 국외 저장 명시 |
| §21 보존·파기 | ⚠️ 기준 없음 | 보존기간·파기 방법 결정·명시 |

| §39-x 안전조치 | ✅ 토큰 서명·deviceId HMAC | `typingverse-token`은 XSS 시 탈취 가능(개인정보 아님이나 보안 참고) |
| 쿠키/동의 | ✅ 쿠키·트래커 없음 | 배너 불필요 |

## 3. 결정이 필요한 사항 (사용자 판단)

**a. 운영주체·연락처** — 처리방침 필수 기재. 개인/사업자 여부, 개인정보 보호책임자 이름·연락처(이메일). 부록의 `[ ]` 칸에 채울 값.

**b. 만 14세 미만(초등) 정책** — 가장 중요한 법적 결정. 셋 중 택1:
   1. **(권장·간단) 개인정보 미수집 포지셔닝** — 실명·이메일 등을 받지 않으므로 "개인을 식별할 수 있는 정보를 수집하지 않는다"고 방침에 명시하고, 닉네임에 실명·연락처 입력을 금지하는 안내를 둔다. 아동 특별 동의 플로우 없이 운영. (현 설계와 가장 부합)
   2. **법정대리인 동의 플로우 도입** — `elem` 선택 시 보호자 동의 절차 추가. 구현 부담 큼.
   3. **만 14세 미만 이용 제한** — 초등 밴드 제거/가입 차단. 교육용 취지와 상충.

**c. 보존기간** — 예: "회원 탈퇴 또는 마지막 접속 후 N개월 경과 시 지체 없이 파기". `ON DELETE CASCADE`가 이미 있어 계정 삭제 시 연관 데이터 자동 삭제됨. 파기 트리거(탈퇴 API/비활성 정리 잡)만 정하면 됨.

## 4. 권고 우선순위

1. **[P1] 개인정보처리방침 페이지 신설** (`/privacy`) — 부록 초안 + 3절 결정값 반영. 푸터에 링크.
2. **[P1] 방침에 국외이전(Cloudflare)·보존기간·아동정책 명시.**
3. **[P2] 닉네임 입력 안내** — "실명·연락처를 닉네임에 넣지 마세요" 문구(아동 포지셔닝 강화).
4. **[P3] 탈퇴/데이터 삭제 기능** — `/api` 계정삭제 엔드포인트(CASCADE 활용). 이용자 권리(열람·삭제) 보장.
5. **[참고] 보안** — `typingverse-token`을 localStorage 대신 httpOnly 쿠키로 옮기는 것은 개인정보가 아닌 보안 개선 항목(선택).
