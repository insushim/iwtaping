## 2. 전략 요약

**포지셔닝**: "한컴타자연습의 재미없음 + 타자연습 사이트들의 낡음"을 동시에 깨는, **초중생~성인 모두를 위한 게임형 타자 플랫폼**. 1차 타깃은 학교(선생님이 학급 단위로 배포)와 10대 — 이미 school-safe 콘텐츠·PWA·APK 기반이 있어 최단 경로.

**핵심 베팅 3개**:
1. **전국 순위 + 리그 승강제** = 사회적 비교가 최강 리텐션 장치 (Duolingo 리그 모델 실증).
2. **학급/그룹 기능** = 교사 1명이 학생 25명을 데려오는 K-교육 유통 구조.
3. **하루 5분 루프** = 일일 퀘스트 3개 + 스트릭 + 시즌 보상으로 "짧게 매일".

**빌드 순서 원칙**: 안정화(P0) → 백엔드 척추(P1) → 순위/리그(P2) → 리텐션 루프(P3) → 리디자인(P4) → 신규 모드(P5) → 상용화 마감(P6). 리디자인을 P4에 두는 이유: 백엔드·순위가 UI 구조(프로필·리더보드·리그 화면)를 새로 요구하므로, 먼저 리디자인하면 두 번 칠하게 됨(P3의 퀘스트·상점 UI도 일부 재도장 감수 — 토큰 기반으로 지으면 비용 최소화). 단 **디자인 토큰 정리(P4a)는 P1과 병행** 가능.

### P1 — 백엔드 척추 (2주) 🔴 중대영역
정적 export를 유지한 채 **Cloudflare Pages Functions(또는 별도 Worker) + D1(SQLite)** 추가. 같은 도메인 `/api/*` 라우팅 → CORS 불필요. Next Route Handler는 export 모드에서 불가하므로 반드시 Pages Functions로 구현. **선결: sw.js `/api/` 캐시 제외(B8)**.

**무료 티어 실산정(교차검증 교정)**: D1은 5GB가 계정 총량이며 실병목은 **일 10만 rows written 쿼터**(인덱스 갱신 포함) — 점수 제출량 설계 시 이 축으로 계산. **KV 무료 쓰기는 일 1,000회뿐이라 '60초 리더보드 캐시'는 무료 티어에서 불성립** → 리더보드 캐시는 **Cache API(무제한·무료) 또는 Worker 메모리 캐시**로 대체. Workers 요청 10만/일 = 초당 1.16req 수준이므로 "MAU 수만 무료"는 트래픽 모델링 후에만 단정(초기 소프트런칭 규모는 무료로 충분).

**계정 모델 — "익명 우선"(아동 서비스 법적 리스크 회피)**:
- 가입 없음. 첫 실행 시 `deviceId`(UUID) 생성 → 서버에서 서명된 토큰 발급.
- 닉네임 + 아바타(마스코트)만으로 전국 순위 참가. **이메일·생년월일·실명 일절 수집 안 함** → 수집 최소화로 리스크 대폭 축소. ⚠️ 단 deviceId·플레이 로그·IP도 결합 식별 가능성이 있으면 개인정보보호법상 "개인정보"에 해당할 수 있어, **만14세 미만 법정대리인 동의 의무(제22조의2)가 완전히 소거되는 것은 아님**(Gemini 교차검증 지적). 대응: ① 개인정보처리방침에 항목·목적 투명 고지 ② 학급 경로는 학교/교사를 통한 동의 수집 절차 문서화(교육기관 관행) ③ **법률 검토는 P6이 아니라 P1 착수 시점에 1회**(deviceId·플레이로그·학급관계 처리가 P1부터 시작되므로). 익명화 수준을 높일수록 유리하나 "동의 불요"라고 단정하지 않는다.
- 닉네임 금칙어 필터(한글 비속어 사전) 필수.
- **Sybil 방지**: deviceId는 무한 재발급 가능(localStorage 삭제) → 계정 생성·점수 제출에 rate limit + Cloudflare Turnstile(무료 봇 차단), 신규 계정은 일정 플레이량까지 'provisional'(순위 미반영) 처리.
- 기기 이전: "전학 코드"는 **영숫자 10자리+, 서버 해시 저장, 24h 만료, 시도 5회 제한** (8자리 숫자는 엔트로피 부족 — 교차검증 교정).
- 선택적 소셜 로그인(구글)은 P6에 옵션으로만(성인 사용자 편의).

**스키마 골자**: `users(id, nickname, avatar, created)` · `scores(user_id, mode, kpm, accuracy, keystroke_digest, created)` · `leagues(...)` · `classes(id, join_code, teacher_device)` · `class_members(...)` · `wallets(user_id, coins, xp)` + `transactions(user_id, type, amount, desc, ts, idempotency_key)`.

**코인/XP 경제 서버 이관(중요)**: 현재 XP·코인은 localStorage 순수 클라이언트라 콘솔 편집으로 무한 획득 가능 — 상점(P3)·순위와 엮이는 순간 서버 원장으로 이관. financial-saas 4단계 안전망 적용: ① 모든 재화 변경은 `transactions` audit log 의무 ② 절대값 덮어쓰기 금지(증분 연산만) ③ 클라이언트 lock + 서버 idempotency key 2단계 ④ 캐시 헤더 rule 순서 점검.

**부정행위 방지(순위 신뢰성 = 서비스 생명)**:
1. 클라이언트가 세션 keystroke 타임스탬프 로그(압축)를 점수와 함께 제출.
2. 서버 검증: 총 시간 대비 타수 물리 한계(>1,300타/분 즉시 반려), 키 간격 분산(봇=간격 균일), 텍스트 해시 대조(제시문 변조 방지), 동일 세션 재제출 방지(nonce).
3. 통계적 이상치는 "검토 대기" 플래그 → 순위 보류. **한계 인정**: 적대적 클라이언트(인간형 지터 주입·물리한계 이하로 조작)는 웹 클라이언트만으론 원천 차단 불가 — 목표는 자동화 치팅 비용 상승 + 상위권 집중 모니터링(리플레이 공개 = 사회적 검증)이며, 완벽 차단을 주장하지 않는다.
4. 스키마 보강: `scores`에 `nonce/text_version/verify_status/hold_reason/retention_until` 필드, keystroke 로그는 압축 후 R2 참조(P5 고스트 레이스에서 재사용).
- **완료 게이트**: 중대영역이므로 FULL 교차검증 + 부하 테스트(순위 조회 p95 < 300ms).

### P2 — 전국 순위 & 리그 (2주) 🔴 중대영역
- **전국 순위**: 일간/주간/전체 × 모드별(속도·정확도·게임별 하이스코어) × 급별(초·중·고·일반 — 자기신고 학급). 리더보드 캐시는 Cache API 60초(KV 아님 — 무료 쓰기 쿼터 문제, P1 참조)로 D1 부하 차단.
- **리그 승강제(Duolingo 모델)**: 주간 30명 버킷, 브론즈→실버→골드→다이아→챌린저. 상위 7명 승급, 하위 5명 강등. "이번 주 남은 시간" 카운트다운 = 마감 압박 리텐션.
- **학급 대항전**: 교사가 코드 발급 → 학생 입장 → 학급 평균 타수 순위(전국 학급 리더보드). 주간 학급전 = 교실 내 자발적 경쟁.
- **내 위치 강조 UI**: 전체 1등보다 "내 앞 3명"을 보여주는 근접 순위가 동기부여 실증 우위.
- 숨겨진 `/ranking` 페이지를 리그 허브로 재구축, 네비 복귀.
- 운영 설계: 리그 마감시각(KST 고정)·동점 규칙(동점 시 먼저 도달자 우선)·주간 정산 Cron의 idempotency(중복 실행 방지 키)·버킷 배정 잠금 명세.

### P3 — 리텐션·중독 루프 (2주)
심리 루프: **트리거(푸시/스트릭 위기) → 행동(5분 연습) → 가변 보상(상자·XP·순위 변동) → 투자(마스코트 성장·리그 잔류)**.

- **마스코트 상점 확장**: 이미 있는 coins/액세서리를 실 상점화 — 모자·배경·이펙트·타건음 테마 구매. 순위 화면에 아바타 노출 = 꾸미기의 사회적 가치.
- **시즌(8주)**: 시즌 한정 뱃지·아바타, 시즌 종료 시 리그 부분 리셋 = 신규·복귀 유저 재출발 기회.
- **성장 가시화**: 주간 리포트("지난주보다 +23타"), 손가락별 히트맵·취약 키 맞춤 드릴(`fingerAccuracy`/`problemKeys` — ⚠️ B2 실측: 현재 수집 배선이 끊겨 있어 **P0에서 AccuracyTracker 재배선이 선행 조건**).
- ⚠️ 아동 서비스 윤리 가드: 확률형 보상은 **무료 재화만**(현금 구매 불가), 확률 표기, 일일 플레이 리마인더는 하루 1회 캡.

2. **static export ↔ Functions 통합 마찰** → 실위험은 flatten 스크립트가 아니라 **sw.js의 `/api/*` cache-first(B8)와 무료 쿼터 산정**. P1 첫 3일 스파이크로 배선 검증, 실패 시 Worker 분리 도메인 플랜 B.

4. **아동 개인정보** → 익명 설계로 원천 축소. 소셜 로그인은 성인 확인 후에만.
5. **콘텐츠 저작권**(§1.4) → P0 전수 교체로 해소, 이후 신규 지문은 "퍼블릭 도메인/자체 창작만" 파이프라인 규칙화.
6. **1:1 실시간 대결 복잡도** → P5 최후순위 + 고스트 레이스로 "대결 감각" 선제공(비동기라 리스크 낮음).
